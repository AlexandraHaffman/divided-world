# Schema Master — форматы данных «Разделённого мира»

Актуальные форматы файлов в `data/`. Файлы читаются фронтендом напрямую
(GitHub Pages), поэтому любые изменения формата должны оставаться
обратно-совместимыми: новые поля добавляем, старые не переименовываем.

## Персонаж — `data/characters/<slug>.json`

Ключевые поля (полный пример — `data/character.schema.json`):

| поле | тип | описание |
|---|---|---|
| `name` | string | Каноническое имя. **Служит идентификатором в связях.** |
| `status` | string | `Активен` / `Мёртв` / `Неизвестно` |
| `faction` | string | Название фракции — как в `data/factions.json` |
| `subfaction` | string | Подразделение — как в `subfactions` фракции |
| `role` | string | Должность/положение (показывается в карте окружения) |
| `threat_level` | number | Уровень угрозы |
| `stats` | object | `intelligence, combat, influence, meta_power, cruelty, will, stealth, unpredictability` (0–11) |
| `card_quote` | string | Цитата на карточке |
| `avatar_web` | url | Мини-аватар (карточки, связи) |
| `avatar_web_full` | url | Крупный портрет (досье, центр карты окружения) |
| `tier` | string | `common / rare / epic / legendary / divine` |
| `relations` | array | Связи персонажа — см. ниже |

### Поле `relations`

```json
"relations": [
  { "to": "Ульяна Спасская", "type": "family", "label": "сестра-близнец" }
]
```

- `to` — **точное** имя персонажа, как в поле `name` его файла;
- `type` — один из `love` (партнёр), `family` (кровь), `ally` (союзник),
  `enemy` (враг);
- `label` — короткая подпись под карточкой связи, необязательна.

Правила:

1. Связи в идеале взаимны: у А есть Б → у Б есть А. Односторонние связи
   допустимы, когда это лор (Дариан не знает о существовании Элари), —
   валидатор их не запрещает, а только перечисляет.
2. Типы могут расходиться (безответная любовь: у него `love`, у неё
   `ally`) — карта рисует связь со стороны того, чьё досье открыто.
3. Проверка: `python3 scripts/validate_relations.py` — падает только на
   реальных ошибках (несуществующее имя, неверный тип, дубль, связь с
   самим собой). Автоматически запускается GitHub Action'ом при пуше
   изменений в `data/` — результат виден во вкладке Actions прямо с
   телефона.

## Фракции — `data/factions.json`

```json
{
  "factions": {
    "Тенебрион": {
      "crest": "data/images/factions/tenebrion.jpg",
      "description": "Теократический абсолютизм…",
      "subfactions": {
        "Церковь Тенебрии": { "description": "…", "emblem": null }
      }
    }
  }
}
```

- `crest` — путь к гербу от корня сайта; если его нет, карта окружения
  рисует монограмму в цвете фракции;
- `subfactions[…].emblem` — необязательная эмблема подразделения
  (тот же формат пути), пока ни у кого не задана;
- `description` подфракции показывается в правой колонке карты окружения.

Цвета фракций живут в `FACTION_COLORS` (`characters/dossier.js`) — это
авторитетный источник, `faction_color` в JSON персонажа вторичен.
